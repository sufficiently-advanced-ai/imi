## Template Usage

Copy this template to create new Architecture Decision Records (ADRs):

1. Copy this file to `ADR-NNN-title.md` where NNN is the next sequential number
2. Update the title and status
3. Fill in the Context section with the problem or issue
4. Document the Decision made
5. List the Consequences (both positive and negative)

## ADR Guidelines

- Keep ADRs concise but complete
- Focus on the "why" behind decisions
- Include alternatives considered
- Update status when decisions change
- Link related ADRs when applicable

## Examples of Good ADR Titles

- ADR-001: Use Architecture Validation Tools for Code Quality
- ADR-002: Implement Layered Architecture Pattern
- ADR-003: Adopt Microservices for New Features
- ADR-004: Use Domain-Driven Design for Entity Modeling
